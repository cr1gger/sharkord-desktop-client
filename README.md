# Sharkord Client

Неофициальный десктоп-клиент (Electron) для self-hosted чат-сервера
[Sharkord](https://sharkord.com). Оборачивает веб-приложение вашего сервера в
нативное окно с треем, поддержкой нескольких серверов и deep-link'ов.

## Возможности

- Приветственный экран и мастер подключения по URL/IP.
- Проверка доступности сервера перед подключением (с понятными ошибками).
- Последний сервер запоминается навсегда и открывается при следующем запуске.
- Несколько серверов с произвольными **именами**, переключение и управление списком.
- Иконка в системном трее Windows: **Показать**, **Сменить сервер**, **Выход**.
  Закрытие окна крестиком сворачивает приложение в трей.
- Deep-link через протокол `sharkord://` — пригласительные ссылки открывают
  приложение из браузера / Telegram / любого места.
- Работают голос, видео и демонстрация экрана (WebRTC): выданы разрешения на
  камеру/микрофон, есть адаптивное окно выбора экрана/окна для screen sharing.
- **Автообновление** из GitHub Releases (electron-updater): при запуске апдейт
  ставится до открытия сервера (полноэкранный экран прогресса); вышедший во время
  работы — тихо скачивается и предлагается кнопкой в баннере (Windows/Linux).
- Сборка **инсталляторов под Windows (.exe), macOS (.dmg) и Linux (AppImage)** —
  одновременно в CI по пушу тега.

## Запуск в режиме разработки

```bash
npm install
npm run gen-icons   # один раз: генерирует иконки из assets/icon.svg
npm start
```

## Сборка инсталляторов

Настоящие инсталляторы под все три ОС собираются автоматически в CI (electron-builder,
матрица `windows / macos / ubuntu`) — см. «Как выпустить релиз». Локально можно собрать
инсталлятор под текущую ОС:

```bash
npm run dist        # NSIS .exe (Windows) / .dmg (macOS) / AppImage (Linux) → release/
```

Для быстрого локального теста под Windows есть портативная сборка (без установщика):

```bash
npm run pack:portable   # dist/Sharkord-win32-x64/Sharkord.exe
```

> На Windows `npm run dist` **не проходит на машине с выключенным Developer Mode**:
> electron-builder падает на распаковке `winCodeSign` (симлинки). Это только локальное
> ограничение — в CI (GitHub-раннеры) сборка идёт нормально. Для локального установщика
> включите Windows Developer Mode или запускайте сборку от админа.

## Автообновление (через GitHub Releases)

Клиент обновляется из **GitHub Releases** штатным механизмом **electron-updater** —
сервер Sharkord для этого не нужен. Источник — `latest*.yml` из последнего релиза
репозитория (`package.json → build.publish`). Два сценария:

- **При запуске** — сначала полноэкранный экран «Проверка / Загрузка / Установка
  обновления», апдейт скачивается и ставится **до** открытия страницы сервера. Если
  обновлений нет (или проверка не удалась / вышел таймаут) — сразу открывается сервер.
- **Во время работы** — раз в ~30 минут идёт фоновая проверка. Новую версию клиент
  **молча скачивает, но не ставит**: внизу появляется баннер «Обновление готово» с
  кнопкой **Обновить**. Нажатие — установка с перезапуском; если просто перезапустить
  клиент, скачанное поставится само (`autoInstallOnAppQuit`).

Платформы: **Windows / Linux** — полноценно; **macOS** — авто-обновление недоступно
(Squirrel.Mac требует платной подписи Apple), `.dmg` качается со страницы релизов
вручную. Выключить авто-проверку — меню **Сервер → Обновлять автоматически**; ручная
проверка (покажет баннер с кнопкой) — **Сервер → Проверить обновления**.

### Как выпустить релиз

По пушу тега срабатывает GitHub Actions (`.github/workflows/release.yml`): собирает
инсталляторы на Windows, macOS и Linux одновременно и публикует один GitHub Release со
всеми артефактами и метаданными обновления (`latest*.yml`).

```bash
git tag v1.1.0
git push origin v1.1.0
```

> Проверить сборку под все ОС без публикации можно пушем в ветку `ci/**` или ручным
> запуском workflow — тогда job публикации пропускается.

## Deep-link: как работают пригласительные ссылки

Браузер не может напрямую запустить десктоп-приложение по обычной `https://`
ссылке — для этого нужен собственный протокол. Клиент регистрирует схему
`sharkord://` и понимает такие форматы:

| Ссылка | Что делает |
|--------|-----------|
| `sharkord://open?url=<полный URL сервера с ?invite=CODE>` | Открывает точный адрес (самый надёжный вариант) |
| `sharkord://join?server=<адрес>&invite=<CODE>` | Добавляет/выбирает сервер и применяет инвайт |
| `sharkord://invite/<CODE>` | Применяет инвайт к текущему серверу |

**Как получить рабочую ссылку из веб-инвайта** `http://myserver:4991/?invite=3upl…`:

```
sharkord://open?url=http%3A%2F%2Fmyserver%3A4991%2F%3Finvite%3D3uplTcFhxTGXurswh8kjAB8U
```

(веб-адрес URL-кодируется и подставляется в параметр `url`). Такую ссылку можно
отправлять в чат/Telegram — при клике откроется приложение. Если сервер ещё не
добавлен, клиент предложит подключиться и применит инвайт после подключения.

## Структура проекта

```
src/
  main.js            главный процесс: окно, трей, меню, протокол, IPC
  preload.js         безопасный мост IPC для UI
  store.js           хранение серверов, последнего выбора и настроек (config.json)
  deeplink.js        разбор ссылок sharkord://
  media.js           разрешения WebRTC + пикер демонстрации экрана
  picker-preload.js  preload окна выбора источника экрана
  updater.js         автообновление через electron-updater (GitHub Releases)
renderer/
  index.html/.css/.js  UI: приветствие, подключение, рабочий экран (webview)
  picker.html/.js      окно выбора экрана для демонстрации
assets/
  icon.svg           исходник иконки → генерируются png/ico/icns
scripts/
  gen-icons.js       генерация иконок (png/ico/icns) из icon.svg
  pack-portable.js   быстрая портативная сборка под Windows (dev)
.github/workflows/
  release.yml        матрица сборки под Windows/macOS/Linux + публикация релиза
```
